The Problem

RL is wasteful. Training data is expensive.

Traditional RL
  • Agent explores randomly for thousands of episodes
  • Most early actions are completely useless
  • Reward function designed by hand — takes weeks
  • No prior knowledge of the task
  • GPU hours wasted before first useful behavior
vs
IL-Bootstrapped RL (Our Approach)
  • Start from human demonstration → skip random exploration
  • LLM generates reward function in seconds (Eureka, ICLR 2024)
  • IL warm start = policy already knows roughly what to do
  • RL refines from a good starting point
  • ~2× performance on hard tasks vs RL alone (IBRL paper)

What IL data actually does to RL

Problem

Thousands of GPU hours wasted on useless exploration. Reward function engineered by hand.

Step 1 — IL Warm Start

Behavior clone the video trajectory → policy now roughly knows the motion. Exploration starts in useful space.

+

Step 2 — LLM Reward Function

GPT-4 reads: task description + robot env code → writes reward_fn.py. No human reward engineering needed.

IRL extracts a reward function from demos (not a policy). The policy is trained using that reward by RL. We skip IRL — the LLM writes the reward directly, informed by the IL trajectory.

Technical Depth

LLM reward functions outperform human experts

Prior Art — Eureka (NVIDIA, ICLR 2024)

LLMs write reward code

  • GPT-4 reads raw environment code
  • Generates executable Python reward functions
  • Outperforms human experts on 83% of tasks
  • 52% average normalized improvement
Our Addition

IL trajectory informs the reward

  • LLM also receives the extracted joint trajectory
  • Reward function shaped by actual human motion
  • Warm start + reward = double advantage
  • No IRL needed — GPT-4 replaces it
# Generated by GPT-4 from: "wave both arms" + G1 env code + trajectory.npy
def reward_fn(obs, action, trajectory_ref):
    arm_match = cosine_similarity(obs['arm_pos'], trajectory_ref[t])
    smoothness = -jerk_penalty(action)
    balance = com_stability(obs['base_pos'])
    return 0.6 * arm_match + 0.2 * smoothness + 0.2 * balance
